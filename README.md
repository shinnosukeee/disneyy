## 発生した事象（エラー） 
ディズニーのサイトにスクレイピングが行えない。

## 期待した結果 
このアプリは、自動的にディズニーのレストランのキャンセル待ちを行うことを目的と
して作成したアプリである。 
このアプリの機能は大きく分けて次の３つから構成される。 
１．ユーザ登録機能（フォーム入力、データ保存、バリデーション） 
２．キャンセル検知（Seleniumによる操作、Sidekiqによる定期実行） 
３．メール通知機能（登録確認メール、キャンセル通知メール） 
## 再現手順 
１．リポジトリのクローンを作成 
２．bundle install 
３．rails db:migrate  
rails db:seed 
４．.envファイルを作成。GMAIL_USERNAME=Gmailアドレス GMAIL_PASSWORD= 
アプリパスワード を入力 
５．バージョン６以上のredisをインストールし、起動：redis-server 
６． Sidekiq の起動：bundle exec sidekiq 
７．サーバー起動 
## 実行環境 
Rails~>7.2.1 
Ruby~>3.3.5 
SQlite 
Redis ver 6 以上

## 補足資料 
各機能に対し、テストを行った。 
ユーザ登録機能：エラーなし 
キャンセル検知：seleniumの起動は確認。しかし、白い画面のままスタック。他のサイトでのテストを行った結果問題なかった為、disneyのサイト側に問題ありと思
われる。調べたところ、bot 対策を行っておりその対策が必要であると考えられる。Sidekiqによる定期実行は問題なし。 
メール通知機能：登録確認メールに関しては確認、キャンセル通知メールに関しては、検知にエラーが発生している為未確認
